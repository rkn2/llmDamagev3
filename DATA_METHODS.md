# llmDamage v3 — Data Collection Methods

This document records how each attribute group was populated, what sources were used,
and exactly how to replicate or extend the work in future sessions.

The spreadsheet is `Flood_DataInput.xlsx` (sheet: `BuidlingAttributes`, 177 columns).
Per-building HTML pages are generated by `generate_detail_pages.py`; rerun that script
after updating any values.

---

## 1. River Distances

**Attributes:** `distance_from_river_edge`, `distance_from_river_centerline`

### What we did

Queried the **USGS National Hydrography Dataset (NHD) Large-Scale Flowlines** REST service
for all stream/river centerlines within a bounding box around downtown Montpelier, then
projected to UTM 19N (EPSG:32619) and measured the shortest distance from each building
coordinate to the merged river geometry.

**River included:** North Branch Winooski River + Winooski River (the two rivers that
caused the July 2023 flood). Unnamed tributaries were excluded.

**Edge distance caveat:** The NHD does not provide an area polygon (bankfull channel) for
the North Branch Winooski in downtown Montpelier — only a centerline. So:
- `distance_from_river_centerline` = exact geodesic measurement to the NHD centerline
- `distance_from_river_edge` = centerline distance − 5 m (estimated half-width of the
  North Branch Winooski in the downtown reach at normal/bankfull stage)

### How to redo / extend

1. **GeoJSON already saved** at `montpelierContext/nhd_flowlines_montpelier.geojson`
   (downloaded 2026-06-24; should be refreshed if NHD is updated).

2. To re-download for a different bounding box, use this REST endpoint:
   ```
   https://hydro.nationalmap.gov/arcgis/rest/services/nhd/MapServer/6/query
     ?geometry=<west>,<south>,<east>,<north>
     &geometryType=esriGeometryEnvelope
     &inSR=4326
     &spatialRel=esriSpatialRelIntersects
     &outFields=gnis_name,ftype,fcode,reachcode,lengthkm
     &returnGeometry=true
     &f=geojson
   ```
   Layer 6 = Large-Scale Flowlines (1:24k resolution). Layer 9 = NHD Area polygons
   (useful if the river is wide enough to have a polygon representation).

3. The computation script is embedded in the session history. Key steps:
   ```python
   import geopandas as gpd
   from shapely.geometry import Point
   from shapely.ops import unary_union
   from pyproj import Transformer

   gdf = gpd.read_file("montpelierContext/nhd_flowlines_montpelier.geojson")
   rivers = gdf[gdf['gnis_name'].isin(['North Branch Winooski River', 'Winooski River'])]
   rivers_m = rivers.to_crs(epsg=32619)
   river_line = unary_union(rivers_m.geometry)

   transformer = Transformer.from_crs("epsg:4326", "epsg:32619", always_xy=True)
   x, y = transformer.transform(lon, lat)   # note: lon first!
   dist_m = Point(x, y).distance(river_line)
   ```

4. **Computed distances** (centerline | edge) are also saved at
   `montpelierContext/building_river_distances.json`.

5. **To get true edge distances:** download the NHD Waterbody polygon layer (layer 12)
   or the FEMA NFHL channel/floodway polygon. For the North Branch Winooski in downtown,
   the NHD Waterbody layer returned only 1 feature (FCode 46006) covering a small reach
   and did not span all 5 buildings. A better source would be the Vermont ANR hydrography
   polygon layer (see §8 below).

## 2. USGS High Water Mark (HWM) Data

**Attributes:** `flood_height_building` (partial — used as a proxy)

### What we did

Used the USGS July 2023 HWM survey data already downloaded at:
`montpelierContext/USGS Highwater Data/table_JulyHWMs.csv`

Filtered to the **North Branch Winooski River** stations in Montpelier (prefix `MM`):
- **MM-737** — School St / N. Branch bridge — elev 526.30 ft, 1.41 ft above grade
- **MM-738** — State St / Main St intersection — elev 526.46 ft, 2.27 ft above grade
- **MM-739** — Near Main St rotary — elev 526.33 ft, 1.70 ft above grade
- **MM-740** — Spring St / Integrative Family Medicine parking lot — elev 526.17 ft
- **MM-741** — Elm Street Cemetery — elev 526.41 ft, 2.30 ft above grade
- **MM-742** — House on Elm Street — elev 528.28 ft, 0 ft above grade

Each building was assigned the nearest HWM station. `flood_height_building` was filled
with the LLM-estimated water depth at the building (from photo analysis) supplemented by
the nearest surveyed HWM elevation.

### What's still needed

`flood_height_building` in the spreadsheet is defined as "based on point cloud" — so the
true value requires a point cloud or LiDAR-derived water surface elevation. The LLM
estimate and USGS proxy are approximations only.

**To get the real value:** The USGS and FEMA produced post-flood inundation maps for the
2023 Montpelier flood. The Vermont DEC or VCGI (Vermont Center for Geographic Information)
may have LiDAR or flood-depth rasters for this event.

## 3. FEMA Flood Zone

**Attributes:** `FEMA_floodzone`

All 5 buildings are in **Zone AE** (1% annual chance / Special Flood Hazard Area) along
the North Branch Winooski in downtown Montpelier.

### How to verify / download

FEMA NFHL is available via the **Map Service Center**: https://msc.fema.gov/portal/home
- Search by address to verify the exact parcel designation
- The NFHL REST service endpoints changed recently; the direct county-level download URL
  (`msc.fema.gov/portal/downloadProduct?productTypeID=NFHL&countyCode=50023`) returned
  "not found" in our session
- Washington County VT FIPS code: **50023**
- Alternative: VCGI (vcgi.vermont.gov) hosts the Vermont NFHL as a statewide layer

## 4. NRHP / Historic District

**Attributes:** `NRHP_ref_number`, `national_register_listing_year`, `located_in_historic_district`,
`building_name_listing`, `sub_national_heritage _list`, `property_of_local_significance`,
heritage value fields

### What we know

All 5 buildings are within the **Montpelier Historic District**, listed on the National
Register of Historic Places on **November 3, 1978**, amended April 19, 1989, and updated
with a 2017 amendment (the PDF at `montpelierContext/National Register of Historic Places-
Downtown Montpelier update (2017).pdf`).

### What we read

We read through PDF pages 1–80 of the 246-page NR document. By page 80 we had covered
resources #1–172, reaching Main St addresses up to 180 Main. The 5 buildings we need
are likely:
- **40 Main St** — even-numbered side of Main St, resource # probably between 172–200
- **100 Main St** — further along the even side, resource # probably 180–220 range
- **27 Langdon St** — Langdon St entries, resource # not yet found
- **112 State St** — State St commercial, likely in the 300–400 range
- **54 Elm St** — Elm St entries, likely in the 250–320 range

### How to find individual resource numbers

**Option A — Continue reading the PDF:** resume at page 81. The inventory continues
sequentially. Even-numbered Main St addresses appear after the odd addresses.

**Option B — NPS database:** go to
`https://www.nps.gov/subjects/nationalregister/database-research.htm` and search by
address. Each individual resource in a district has its own record.

**Option C — VCGI / Vermont Division for Historic Preservation:** Vermont publishes the
NRHP data as a GIS layer at vcgi.vermont.gov (search "National Register Historic Places").
This layer has resource numbers and addresses as attributes — faster than reading the PDF.

## 5. Archetype

**Attribute:** `archetype` (integer 1–15)

### Classification system

From Nofal & van de Lindt (2020), *Water* 12(8):2277. There are 15 archetypes (F1–F15),
entered as integers 1–15 in the spreadsheet. The guide instructs to match the building to
the closest type from visual/structural inspection; notes uncertainty in the notes column.

| # | Name | Description |
|---|------|-------------|
| 1 | F1 | One-story single-family residential, crawlspace |
| 2 | F2 | One-story multi-family residential, slab-on-grade |
| 3 | F3 | Two-story single-family residential, crawlspace |
| 4 | F4 | Two-story multi-family residential, slab-on-grade |
| 5 | F5 | Small grocery store / gas station with convenience store |
| 6 | F6 | Multi-unit retail building (strip mall) |
| 7 | F7 | Small multi-unit commercial building |
| 8 | F8 | Super retail center |
| 9 | F9 | Industrial building |
| 10 | F10 | One-story school |
| 11 | F11 | Two-story school |
| 12 | F12 | Hospital / clinic |
| 13 | F13 | Community center / place of worship |
| 14 | F14 | Office building |
| 15 | F15 | Warehouse (small/large box) |

### What we assigned

None of the 15 archetypes explicitly represents a **multi-story historic masonry downtown
commercial block**. The system was designed for community-level resilience analysis across
diverse building stock, not specifically for historic URM buildings.

| Building | Archetype | Rationale |
|---|---|---|
| 100 Main St | **7** (F7) | Multi-tenant commercial (bar/restaurant + upper floors); F7 is closest for small multi-unit commercial |
| 112 State St | **14** (F14) | Bank/office primary use; F14 (office building) is the closest match |
| 27 Langdon St | **7** (F7) | Multi-tenant retail/commercial |
| 40 Main St   | **7** (F7) | Hardware + restaurant + copy shop = multi-unit retail; F7 closest |
| 54 Elm St    | **7** (F7) | Laundromat (mercantile) + likely residential upper floors; F7 closest |

All with certainty 1 (0–25%) — the mismatch on stories and construction type is real.

### Key limitation to document

The Nofal & van de Lindt portfolio does not include a multi-story masonry commercial
archetype. This is a known gap when applying this system to historic downtown buildings.
Should be flagged in any fragility analysis that uses these archetype assignments.

## 5b. Wall Thickness

**Attribute:** `wall_thickness` (meters)

The guide says "use 0.2 m (assumed)" for typical residential or similar buildings. This
is the default for wood-frame residential — **not appropriate for historic masonry**.

For late 19th century commercial masonry buildings in downtown Montpelier:
- Typical range: 12–24 inches (0.30–0.61 m) for load-bearing brick walls
- 3-story buildings often have ~18-inch (0.46 m) walls at base
- We used **0.46 m** as a mid-range assumed value; certainty 1

To refine: look for damage photos showing wall cross-section, or consult historic
structural drawings if available through Vermont Division for Historic Preservation.

## 6. LLM Flood Damage Assessment

**Attributes:** `_llm_damage_level`, `_llm_confidence`, `_llm_water_depth_ft`,
`_llm_reasoning`, `_llm_limitations`, `status_u`, `degree_of_damage_u`, etc.

These come directly from `address_assessments.json`, output of the llmDamagev3 pipeline
(`run_pipeline.py`). The model (claude-sonnet-4-6) receives only before/after photos and
the address — no ground truth, no HWM data, no historic context.

## 9. Generator Script

`generate_detail_pages.py` — run this to rebuild all 5 HTML detail pages after updating
any attribute values. The script:
1. Reads the 177-column schema from `Flood_DataInput.xlsx` (BuidlingAttributes header)
2. Reads attribute definitions and valid options from the AttributeInfo sheet
3. Injects computed river distances from `RIVER_DISTANCES` dict at the top of the script
4. Outputs one HTML file per building to `building_details/`
5. Writes `building_detail_manifest.json` (used by `results.html` for address links)

To add a new computed value: update the relevant building's dict in `BUILDINGS` and rerun.
To add a new attribute not in the 177 columns: add it to `BUILDINGS` and to `SECTIONS`
in the appropriate section.

## 10. Auto-Collected Building Attributes

**Script:** `collect_building_attributes.py`
**Output:** `building_attributes_auto.json`

Runs three API queries per building and writes results to JSON:

| Attribute | Source | Notes |
|---|---|---|
| `latitude`, `longitude` | OSM Nominatim geocoding | 1 req/s rate limit |
| `ground_elevation_m` | USGS 3DEP EPQS (bare-earth LiDAR, NAVD88) | Not the same as `first_floor_elevation_m` — add step height |
| `building_area_m2` | OSM Overpass building polygon (Shoelace formula, UTM-local projection) | |
| `approx_wall_length_a_m` | N-S bounding-box extent of footprint | Assign to front or side based on street orientation |
| `approx_wall_length_b_m` | E-W bounding-box extent of footprint | |
| `osm_building_type`, `osm_name`, `osm_levels`, `osm_height` | OSM tags (present only when OSM has them) | Height/levels not populated for these buildings |

**Resolved caveat:** 100 Main St and 27 Langdon St originally returned the same OSM polygon (their
geocoded points are only ~25 m apart, so Overpass picked the same nearest building way for both).
Cross-checked against Vermont's statewide E911 address points and tax parcel boundaries
(VTrans `ROW/Parcels` ArcGIS REST service — see Section 8 sources) and found they are in fact
two separate buildings/parcels: 100 Main St is its own parcel (095.100000, owner City Line Realty
LLC); 27 Langdon St is a unit address inside the building whose primary E911 address and parcel is
90 Main St (095.090000, owner Malone 210 College Street Properties LLC). `building_attributes_auto.json`
now uses the 90 Main St parcel boundary (811.2 m², ~43.0 × 38.5 m bounding box) for 27 Langdon St
instead of the shared OSM figure.

See Section 7 for the full list of attributes still requiring visual/manual entry.
